How the Policy Engine Works
1. Command normalisation — strips path prefixes, lowercases
2. Blocked-command lookup — exact match against deny list
3. Sensitive-path check — argument scanning for /etc/shadow etc.
4. Risky-flag scoring — -rf, --force, URL patterns add risk
5. Risk threshold gate — cumulative score ≥ 80 → auto-block
6. Audit-list match — known-safe commands are logged + allowed
7. Default sandbox — unknown commands run in restricted jail
